# soap_descriptors

This module computes SOAP descriptors for frames generated by AIMD.

本模块用于为 AIMD 轨迹中的结构帧计算 SOAP 描述符，并将结构转换为后续机器学习与降维分析可使用的数值特征。

## Goal

- Convert structures into fixed-length SOAP vectors
- Save descriptor matrices with frame identifiers
- Prepare data for PCA, clustering, and novelty analysis

## 模块目标

- 将结构转换为固定长度的 SOAP 特征向量
- 保存描述符矩阵及对应帧编号
- 为 PCA、聚类、结构新颖性分析和神经网络筛选提供输入

## What Is SOAP

SOAP means `Smooth Overlap of Atomic Positions`.

It is a structural descriptor that converts the local atomic environment into a numerical vector. The main purpose is to represent structures in a way that is more stable and more suitable for machine learning than raw Cartesian coordinates.

## SOAP 的含义

SOAP 是 `Smooth Overlap of Atomic Positions` 的缩写，本质上是一种结构描述符。

它的作用是把原子局域环境表示成数值向量，使结构能够以统一的特征形式输入到机器学习模型中，而不是直接使用原始坐标。

相比直接使用原子坐标，SOAP 的优点是：

- 更适合比较不同结构之间的相似性
- 更适合做降维、聚类和筛选
- 对结构表示更稳定，不依赖简单的坐标排列方式

## Role In This Workflow

SOAP is the representation layer of the active-learning workflow.

It does not decide which structures are important by itself. Instead, it converts each frame into a machine-readable feature vector that downstream modules can use.

## 在这条流程中的作用

在你的流程里，SOAP 处于“结构表示层”。

它本身不是筛选器，而是将轨迹帧转换为统一特征，供后续模块使用，例如：

- PCA 可视化
- 聚类分析
- 新颖结构识别
- 神经网络结构筛选器
- 最终训练集构建

因此流程关系可以理解为：

```text
AIMD trajectory
  -> SOAP descriptor
  -> PCA / clustering / neural-network screening
  -> final training dataset
```

## Expected inputs

- AIMD trajectory frames
- SOAP hyperparameters

## 输入说明

- AIMD 输出的结构帧，通常来自 `AIMD_dataset/cif_frames/`
- SOAP 超参数配置，例如：
  - `species`
  - `rcut`
  - `nmax`
  - `lmax`
  - `sigma`

## Expected outputs

- `outputs/descriptors.npy`
- `outputs/frame_index.csv`
- `outputs/metadata.json`

## 输出说明

- `outputs/descriptors.npy`
  保存所有结构帧的 SOAP 向量矩阵
- `outputs/frame_index.csv`
  记录每一行 descriptor 对应哪个帧文件
- `outputs/metadata.json`
  记录 SOAP 参数、输入来源和运行信息

## Minimal workflow

1. Read frames
2. Build SOAP generator
3. Compute descriptors per frame
4. Save arrays and frame mapping

## 逻辑说明

1. 读取 AIMD 结构帧
2. 根据配置初始化 SOAP 生成器
3. 对每一帧或每个局域环境计算 descriptor
4. 将结果保存为矩阵和索引映射文件

## 配置说明

配置文件为 [config.yaml](./config.yaml)。

当前参数示例：

```yaml
soap:
  species: ["C", "H", "O", "Zn"]
  rcut: 5.0
  nmax: 8
  lmax: 6
  sigma: 0.5
  periodic: true
```

这些参数控制 SOAP 描述符如何表征结构局域环境。

## Notes

- SOAP is a feature generator, not a selector by itself.
- PCA is a downstream analysis of SOAP features, not a replacement for SOAP.
- The current folder is still a scaffold and does not yet implement descriptor generation.

## 备注

- SOAP 是特征生成模块，不直接负责筛选结构。
- PCA 是 SOAP 之后的下游分析步骤，不是 SOAP 的替代品。
- 当前该模块仍是骨架，真实的 descriptor 计算逻辑还没有正式实现。
