# nn_uncertainty

This module trains and applies a standalone neural-network structure selector.

本模块不再表示“模型不确定性评估器”，而是一个独立的神经网络结构筛选器，用来判断哪些结构值得保留进入候选训练集。

## Goal

- Load structure descriptors and auxiliary labels
- Train a standalone neural-network selector
- Score all candidate structures
- Export high-value structures for the final dataset

## 模块目标

- 读取结构描述符和辅助标签
- 训练一个独立的神经网络筛选器
- 对所有候选结构进行打分
- 导出值得保留的高分结构

## Module Position

This module is a structure selector, not a potential-model uncertainty estimator.

## 模块定位

这个模块的目标不是比较多个势函数模型之间的分歧，也不是做传统 committee uncertainty。

它的真正定位是：

- 输入结构特征
- 输出结构筛选分数

也就是说，它学习的是“哪些结构值得保留”，而不是“模型对哪些结构不确定”。

## Logic

1. Load descriptors generated by `soap_descriptors`
2. Load optional labels or heuristic targets from upstream analysis modules
3. Train a neural-network classifier or regressor
4. Score each structure frame
5. Select structures above a threshold
6. Export scores, manifests, and chosen frames

## 逻辑说明

1. 读取 `soap_descriptors` 生成的结构特征
2. 读取来自上游规则模块的辅助标签或目标分数
3. 训练一个神经网络分类器或回归器
4. 对每个结构帧进行打分
5. 根据阈值筛选高价值结构
6. 导出评分表、候选结构清单和关键帧

## Possible Labels Or Targets

- Bond-change frames from `bond_change_detection`
- Zn-O coordination change frames from `zn_o_coordination`
- Rare or novel structures identified in feature space
- Weighted importance scores combined from multiple rules

## 可用标签或目标来源

- `zn_o_coordination` 识别出的配位变化帧
- 特征空间中稀有或新颖的结构
- 由多种规则加权得到的重要性分数

**这段神经网络本质上学的是什么**
学的是一个映射：

```text
SOAP descriptor  ->  selection_score
```

更具体地说，是：

```text
结构特征 -> 这个结构是不是“值得保留”
```

而不是：

```text
结构 -> 势能
结构 -> 力
```

所以这是结构筛选模型，不是势函数模型。

**模型结构本身**
模型定义在 [run.py#L36](/c:/Users/yingkaiwu/Desktop/Active-learning/NNP-reference/scripts/nn_uncertainty/run.py#L36)。

```python

```

意思是定义一个 PyTorch 神经网络类，继承 `nn.Module`。

网络结构是：

- 输入层：`input_dim`
- 隐藏层 1：`Linear(input_dim, 256)` 之类
- `ReLU`
- `Dropout`
- 隐藏层 2
- `ReLU`
- `Dropout`
- 隐藏层 3
- `ReLU`
- 输出层：`Linear(prev_dim, 1)`

代码里通过循环动态构造：

```python
for hidden_dim in hidden_dims:

    layers.append(nn.ReLU())
```

这表示你在配置里写多少层，它就建多少层。

**训练过程**
在 [run.py#L207](/c:/Users/yingkaiwu/Desktop/Active-learning/NNP-reference/scripts/nn_uncertainty/run.py#L207)。

标准 PyTorch 训练循环是：

```python
for epoch in ...:
    model.train()
    for batch_x, batch_y in train_loader:
        optimizer.zero_grad()
        logits = model(batch_x)
        loss = criterion(logits, batch_y)
        loss.backward()
        optimizer.step()
```

含义分别是：
- `model.train()`：切换到训练模式，启用 dropout
- `zero_grad()`：清空上一轮梯度
- `model(batch_x)`：前向传播
- `loss = ...`：计算损失
- `loss.backward()`：反向传播，算梯度
- `optimizer.step()`：用梯度更新参数

这是神经网络训练最核心的一段。

**为什么用 `BCEWithLogitsLoss`**
在 [run.py#L203](/c:/Users/yingkaiwu/Desktop/Active-learning/NNP-reference/scripts/nn_uncertainty/run.py#L203)。

因为现在是二分类：
- 正样本 = 值得保留
- 负样本 = 普通结构

`BCEWithLogitsLoss` 比“先 sigmoid 再 BCE”更稳定。

也就是说训练时模型输出的是 `logits`，不是概率。  
只有在最终打分时，才手动做：

```python
scores = torch.sigmoid(logits)
```

见 [run.py#L240](/c:/Users/yingkaiwu/Desktop/Active-learning/NNP-reference/scripts/nn_uncertainty/run.py#L240)

**为什么有 `pos_weight`**
在 [run.py#L197](/c:/Users/yingkaiwu/Desktop/Active-learning/NNP-reference/scripts/nn_uncertainty/run.py#L197)。

通常正样本少、负样本多，所以这里加了类别不平衡权重：

```python
pos_weight = negative_count / positive_count
```

作用是让模型别只会预测 0。
